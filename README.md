# 张雪峰高考专业选择 Skill

> **帮你选择高考专业以及后续人生发展路线。基于张雪峰老师B站200+教学视频ASR转写分析提炼而成。**

本 Skill 将张雪峰老师公开分享的高考志愿填报方法论系统化提炼为可复用的决策框架。语料总量 298,367 字符，覆盖 114 个视频。

---

## 快速使用

### 安装

```bash
# 方式一：复制到 Claude Code skills 目录
cp -r skills/xue-feng-skill ~/.claude/skills/

# 方式二：直接使用 SKILL.md 作为参考
cat SKILL.md
```

安装后在 Claude Code 中直接问专业选择相关问题即可自动激活，或将 SKILL.md 作为 prompt 输入到其他 AI 工具中使用。

### 支持的使用场景

- 高考志愿填报、专业选择
- 某个专业的就业前景、薪资水平咨询
- 根据自身条件（分数、家庭、性别、地域）选专业
- 考研/考公方向选择
- 纠结两个专业之间如何选择
- 大学毕业后的发展路径规划

## 专业选择分析流程

使用本 Skill 进行专业选择时，遵循以下分析流程：

```
┌─────────────────────────────────────────────────────────────────┐
│                  张雪峰专业选择分析流程                           │
└─────────────────────────────────────────────────────────────────┘

  ┌───────────┐
  │  Step 1   │
  │  信息采集  │
  │           │
  │ 分数/排名  │
  │ 省份/选科  │
  │ 性别/家庭  │
  │ 城市偏好   │
  └─────┬─────┘
        │
        ▼
  ┌───────────┐
  │  Step 2   │
  │  诉求明确  │
  │           │
  │ 优先级排序  │
  │ 核心需求   │
  └─────┬─────┘
        │
        ▼
  ┌───────────┐     ┌───────────┐
  │  Step 3   │     │  Step 4   │
  │  家庭定级  │────▶│  排除法   │
  │           │     │           │
  │ 普通/有资源│     │ 英语短板  │
  │ /资源丰富  │     │ 资源不匹配│
  └───────────┘     │ 行业下行  │
                    └─────┬─────┘
                          │
                          ▼
  ┌───────────┐     ┌───────────┐
  │  Step 5   │     │  Step 6   │
  │  匹配推荐  │────▶│  路径规划  │
  │           │     │           │
  │ 候选专业   │     │ 大学→就业  │
  │ 风险分析   │     │ 3-5年发展 │
  └───────────┘     └───────────┘
```

### 各步骤说明

| Step | 动作 | 说明 |
|------|------|------|
| 1 | 信息采集 | 收集分数、省份、选科、性别、家庭条件、城市偏好等 7 项核心信息 |
| 2 | 诉求明确 | 确认用户最看重什么（就业/稳定/深造/兴趣/地域），明确优先级 |
| 3 | 家庭定级 | 判断家庭属于"普通家庭"、"有一定资源"还是"资源丰富"，决定专业选择范围 |
| 4 | 排除法 | 根据条件排除不匹配的专业（英语短板、资源不匹配、行业下行等） |
| 5 | 匹配推荐 | 从剩余选项中推荐 2-3 个方向，说明适配理由和风险点 |
| 6 | 路径规划 | 给出大学四年行动清单 + 毕业后 3-5 年发展路径 |

## A/B 测试：使用 Skill vs 不使用 Skill

以下是对同一个真实案例的对比分析，展示 Skill 的实际效果。

### 案例背景

| 维度 | 信息 |
|------|------|
| 考生 | 2026年湖北武汉高考男孩 |
| 成绩 | 超武大华科基本线，可冲上交复旦王牌专业 |
| 选科 | 理化生（传统理科） |
| 家庭 | 父亲是东风系供应商中国区经理，母亲全职主妇 |
| 资产 | 多套房产+铺面+良好现金流 |
| 诉求 | 父亲要基础学科，母亲要留武汉，孩子喜欢古币/历史 |
| 短板 | 英语较差 |

### 不使用 Skill（通用回答）

**典型输出**：
> "建议选数学或物理学，这是基础学科，就业面广。武大华科都是好学校，留武汉可行。如果分数够上交复旦也可以考虑。金融也是不错的选择..."

**问题**：
1. 没有系统化的决策框架，东一榔头西一棒子
2. 没有根据家庭资源做匹配分析
3. 没有排除法，推荐了"金融"却没有说明资源门槛
4. 没有考虑英语短板对择校的影响
5. 没有给出后续发展路径规划
6. 没有验证最新就业数据的意识

### 使用 Skill（张雪峰框架）

**典型输出**：

| 步骤 | 动作 |
|------|------|
| Step 1 | 采集 7 项信息，确认完整 |
| Step 2 | 明确三级诉求优先级：P1基础学科 > P2留武汉 > P3兴趣 |
| Step 3 | 定级家庭为"有一定资源的家庭"，可选资源铺路方向 |
| Step 4 | 排除法：排除英语依赖型、资源不匹配型、行业下行型 |
| Step 5 | 三大候选方向 + 风险分析 + 发展路径 |

**优势**：
1. 有完整的决策流程（采集→诉求→排除→匹配→路径）
2. 家庭背景定级准确（"有一定资源"而非"普通家庭"）
3. 排除法清晰（英语差排除外语/国贸，资源不匹配排除管理类）
4. 匹配推荐有理有据（华科物理完美对接东风系新能源资源）
5. 给出大学四年的行动清单和多条发展路径
6. 提醒搜索最新数据验证

### 对比总结

| 维度 | 不用 Skill | 用 Skill |
|------|-----------|---------|
| 决策框架 | 无，随机推荐 | 十大维度 + 五步流程 |
| 家庭分析 | 浮于表面 | 精准定级 + 资源匹配 |
| 排除逻辑 | 无 | 四轮排除，逻辑清晰 |
| 推荐质量 | 泛泛而谈 | 精准匹配 + 路径规划 |
| 风险提示 | 无 | 每个方向都有风险点 |
| 可执行性 | 低（"选数学吧"） | 高（大一到大四行动清单） |
| 先验验证 | 无 | 提醒搜索最新数据 |

**结论**：使用 Skill 后，回答从"正确的废话"升级为"可执行的行动指南"。

## 附录：Skill 构建流程

> 以下内容记录了本 Skill 是如何从原始视频构建而成的，仅供需要复现或改进的开发者参考。

### 构建 Pipeline

```
┌─────────────────────────────────────────────────────────────────┐
│                     Skill 构建 Pipeline                         │
└─────────────────────────────────────────────────────────────────┘

  B站搜索 → 视频下载 → 音频提取 → ASR转写 → 内容分析 → Skill生成
```

| Step | 模块 | 输入 | 输出 |
|------|------|------|------|
| 1 | `bilibili_search` | 关键词 | `search_results.json` |
| 2 | `bilibili_downloader` | 视频列表 | `data/videos/*.m4a` |
| 3 | `audio_extractor` | M4A 文件 | `data/audio/*.wav` |
| 4 | `whisper_transcriber` | WAV 文件 | `data/transcripts/*` |
| 5 | `content_analyzer` | 转写文本 | `analysis_report.json` |
| 6 | `skill_generator` | 分析报告 | `SKILL.md` |

### 从零构建

```bash
pip install -r requirements.txt
python pipeline.py                # 全流程
python pipeline.py --step asr     # 只运行 ASR
python pipeline.py --from-step asr  # 从 ASR 开始到最后
```

### 数据说明

`data/transcripts/` 目录包含 137 个视频的 ASR 转写结果：

- `.txt` — 纯文本，可直接阅读
- `.json` — 包含分段时间戳信息（start/end/text）

转写使用 OpenAI Whisper `base` 模型完成。由于是自动语音识别，部分专有名词可能存在识别误差。
